# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## Commands

```bash
npm run dev       # Start Vite dev server with hot reload
npm run build     # TypeScript check + Vite production build
npm run preview   # Preview production build locally
```

No test framework is configured.

## Environment

Copy `.env.example` to `.env` and set `VITE_ANTHROPIC_API_KEY` — the app calls the Anthropic API directly from the browser and will not load without this key.

## Architecture

A pure-frontend React + TypeScript SPA (no backend) for competitive SIEM vendor intelligence. Vendors are compared across weighted scoring categories; scores can be entered manually or generated by Claude.

**Data flow:**
1. Seed data in `src/data/` seeds Zustand stores on first load
2. Stores persist to localStorage (keys: `ng-siem-vendors`, `ng-siem-categories`, `ng-siem-comparison`)
3. Components read from stores via hooks; mutations go through store actions
4. AI assessments stream from Claude Sonnet 4.6 via `src/services/claudeService.ts`, using tool use to return structured JSON (`submit_vendor_assessment` tool)

**Four Zustand stores** (`src/store/`):
- `useVendorStore` — vendor list and CRUD; recalculates `overallScore` whenever scores/weights change
- `useCategoryStore` — category definitions and weights; calls back into vendor store when categories are added/removed
- `useComparisonStore` — UI state (selected vendors, active view, sort, filters); Sophos is always locked as selected
- `useThemeStore` — light/dark preference applied to `document.documentElement`

**Views** (`src/components/`):
- `comparison/ComparisonMatrix` — primary table view (vendors × categories)
- `charts/RadarChart`, `HeatmapGrid`, `BarComparison` — alternative visualizations
- `layout/AppShell` — shell with `TopNav`, `Sidebar`, `MainContent`
- `ai/StreamingText` — renders Claude's streaming thinking output
- `vendors/AddVendorModal` — add custom vendor; triggers AI assessment flow
- `categories/ManageCategoriesModal` — add/remove/reweight categories

**Scoring:**
- `src/utils/scoreUtils.ts` owns all score math: weighted average, color thresholds (blue ≥9, green ≥7, amber ≥5, red <5), and weight normalization
- Category weights must sum to 1.0; `normalizeWeights()` is called on edits
- `dataSource` on each `CategoryScore` is `'manual' | 'ai-generated' | 'ai-reviewed' | 'unscored'`

**AI integration** (`src/services/claudeService.ts`):
- Calls `client.messages.stream()` with `dangerouslyAllowBrowser: true` — this is intentional; there is no server
- Tool schema is built dynamically from current categories at call time
- `useAIAssessment` hook (`src/hooks/`) manages loading/error state and cancellation via `AbortController`
- Model is hard-coded to `claude-sonnet-4-6`; max tokens 4096

**Business constraints baked into the code:**
- Sophos is always the first vendor and cannot be deselected (`isSophos: true`)
- Maximum 7 vendors in comparison at once
- Built-in categories cannot be deleted (only custom ones can)
